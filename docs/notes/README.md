# Flikky 项目笔记

项目全流程的沉淀：复盘、踩坑、决策、协作经验。目标是让下一次做类似项目时，能以此为起点少走弯路。

## 索引

- [retrospective.md](retrospective.md) — 每个版本完成后的复盘，记录做了什么、跑多快、满意/不满意
- [traps-and-fixes.md](traps-and-fixes.md) — 踩过的坑与修复方案（技术栈陷阱、环境问题、设计缺陷）
- [decisions.md](decisions.md) — 技术与产品决策记录（为什么选 A 不选 B）
- [ai-collaboration.md](ai-collaboration.md) — 用 Claude Code + subagent-driven 开发的经验

## 更新规则

- 每完成一个小版本（v1.0 / v1.1 / ...），追加一节到 `retrospective.md`
- 遇到任何"踩坑 → 修复"的循环，立刻记到 `traps-and-fixes.md`
- 做任何"多方案取一"的决策，同步补到 `decisions.md`，写清理由
- AI 协作的心得随时补到 `ai-collaboration.md`

**一条心法：** 如果某个知识点让我花了超过 10 分钟才搞明白，它就有资格被记下来。
