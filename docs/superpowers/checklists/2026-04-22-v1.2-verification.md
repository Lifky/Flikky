# Flikky v1.2 人工验收清单

> 前置：v1.1 全部通过。设备已连同 WiFi。本版重点是多会话导出 + 上传进度 + snackbar + WiFi 重绑。

## 基本功能（v1.1 回归）

- [ ] 主页显示历史会话列表；FAB 启动服务仍然正常
- [ ] 启动传输服务后浏览器 PIN 登录、发文本 + 文件双向跑通（v1.0/v1.1 功能无回归）
- [ ] 停止服务后主页多出一条会话；点击进入历史详情
- [ ] 长按菜单「置顶 / 重命名 / 删除」仍然工作

## 多会话导出（主干）

### 选择模式

- [ ] 主页 TopAppBar 右上角出现「导出」按钮（文字）
- [ ] 点「导出」→ 卡片左侧出现 Checkbox；FAB 隐藏；底部出现 action bar（"已选 0 个 / 共 N 个"）
- [ ] TopAppBar 变为：左 ✕、中 "已选 N 个"、右 "全选"
- [ ] 进行中会话（未停服）灰掉、Checkbox disabled，副标题含"结束服务后可导出"
- [ ] 勾选若干已结束会话 → action bar 数字同步刷新
- [ ] 点"全选"→ 所有可选会话被勾选（进行中的依然不勾）
- [ ] 物理返回键在多选状态下 → 先退出多选，不退出 APP
- [ ] 点 ✕ → 立即退出多选

### 互斥规则

- [ ] 服务正在跑时点「开始导出」→ Snackbar「请先停止当前服务」
- [ ] 先停服务再导出 → 成功进入 ExportingScreen

### 导出过程

- [ ] 点「开始导出 (N)」→ 进入 ExportingScreen，显示 URL + PIN
- [ ] 通知栏文案区分："正在提供导出: N 个会话 / X MB"（与传输模式不同）
- [ ] 浏览器打开 URL → PIN 登录 → 跳转 `/export` 页
- [ ] `/export` 页显示会话清单（名称 / 消息数 / 文件数 / 总大小）
- [ ] 「下载全部 (X MB)」按钮点击 → 浏览器开始下载 zip
- [ ] 大文件会话（>100 MB）下载中 APP 端 ExportingScreen 看到 LinearProgressIndicator 移动
- [ ] 下载完成后：
  - [ ] 浏览器端文件自动保存
  - [ ] APP 端 ExportingScreen 自动跳 Done 屏
  - [ ] 服务自动停止（通知栏消失）

### 导出完成处理

- [ ] Done 屏显示 ✓ + "已导出 N 个会话到 PC"
- [ ] 点「保留本地」→ 回主页；会话记录与文件都还在
- [ ] 点「删除本地」→ 弹 AlertDialog 二次确认「确认删除本地 N 条会话？」
  - [ ] 点"取消"→ 对话框关闭，回 Done 屏
  - [ ] 点"确认删除"（红色）→ 本地 N 条会话 + 文件目录一起删（用 `adb shell run-as com.example.flikky ls files/sessions/` 验）

### zip 内容

- [ ] 用解压工具打开 zip：根目录有 `README.txt`（含导出时间、会话数、总字节）
- [ ] `sessions/{id}_{name}/` 每个子目录含 `messages.txt`、`messages.json`、`files/`
- [ ] `messages.txt` 行格式 `[时间] [PHONE  ] ...` / `[BROWSER] ...`，文件消息显示 `[文件] name (size)`
- [ ] `messages.json` 能被 `jq .` 解析，schema 正确
- [ ] 会话名含 `:` 字符时目录名里变成 `·`（中点）；其它非法字符（`/ \ * ? " < > |`）变 `_`；超长截断 40 字符
- [ ] 同会话内重名文件自动 `_2` / `_3` 后缀

### 边界场景

- [ ] 只勾 1 个会话也能导出（单会话）
- [ ] 不勾任何会话时「开始导出 (0)」按钮 disabled
- [ ] 导出中点「取消导出」→ 服务立即停；回主页，session 保留
- [ ] 强杀 APP → 重启后 exportMode 清空；主页正常显示

## B1 大文件上传进度

- [ ] 浏览器选 100 MB 文件上传 → 消息列表立即出现 uploading 占位泡（文件名 + 进度条）
- [ ] 进度条随上传前进而刷新（可见百分比变化）
- [ ] 上传完成后占位泡转为正式泡（可点击下载）
- [ ] 浏览器同时上传多个文件 → 每个文件独立占位泡，各自进度
- [ ] 服务端主动触发失败（如停服后上传）→ 占位泡变红边框 + "上传失败，点击重试"
- [ ] 点击重试 → 用相同 File 对象重新上传
- [ ] WS `file_added` 广播不会产生重复泡（已有 fileId 的跳过）

## B2 mdui snackbar

- [ ] 浏览器上传失败 → 弹 snackbar（不再是系统 alert）
- [ ] 导出会话失效等错误 → snackbar
- [ ] Snackbar 样式与 mdui 主题一致（不是浏览器原生对话框）

## WiFi 切换自动重绑

- [ ] 启动服务（绑到 WiFi A 的 IP）
- [ ] 切换到 WiFi B（或手动重连 WiFi）→ APP 顶部出现 banner：
  - [ ] 切换中 → "正在切换网络…"（黄色）
  - [ ] 切换完成 → "网络已切换，URL 已更新为 ..."（蓝色）+ 「我知道了」按钮
- [ ] 点「我知道了」→ banner 消失；主卡片 URL 显示的是新 IP
- [ ] 浏览器用新 URL 打开能连上（旧 URL 自然失效）
- [ ] 关掉 WiFi（断网）→ banner 变 "已失联，等待网络恢复"（红色）
- [ ] 重新连 WiFi（可能 IP 变）→ 回到 "切换中" → "已切换"
- [ ] 同场景下 ExportingScreen 的 banner 也工作

## 安全（回归 v1.0+v1.1）

- [ ] PIN 单次使用、3 次锁 30 秒、5 次终止 —— 无回归
- [ ] 通知栏只显示 URL 不显示 PIN —— 无回归
- [ ] CSP / X-Frame-Options 等 response header 无变化

## 稳定性

- [ ] 上传 100 MB 大文件能成功
- [ ] 连续 3 次导出（每次勾 2 会话）无崩溃
- [ ] 导出期间切 WiFi（极端场景）→ 要么 banner 正确显示 + 浏览器拿到半 zip，要么导出卡住——观察并记录现象，不作为阻塞项
