# Flikky v1.1 人工验收清单

> 前置：v1.0 全部通过。设备已连同 WiFi。

## 基本功能

- [ ] 首次打开 APP（冷启动，DB 空）：显示「尚无历史会话」空态卡片 + 右下 FAB
- [ ] 点 FAB → 进入 ServingScreen；通知栏出现
- [ ] 浏览器输 PIN 进入聊天，发 2 条文本 + 1 个文件
- [ ] 手机点「停止服务」 → 返回主页，主页显示刚才的会话（1 条）
- [ ] 会话副标题含预览（第一条文本前 40 字）
- [ ] 点击会话卡片 → 进入 HistoryScreen，消息气泡按时间渲染
- [ ] 文件气泡点击 → 系统选择器可打开文件
- [ ] 手机自己发的文件气泡在历史里仍只提示「已发送」，不可点击

## 会话管理

- [ ] 主页长按会话 → 弹菜单「置顶 / 重命名 / 删除」
- [ ] 重命名生效且刷新（重启 APP 仍保留）
- [ ] 置顶：卡片加 📌；列表排到最上；非置顶按时间降序
- [ ] 删除：卡片消失；文件系统下 `sessions/{id}/` 目录一起删（用 `adb shell run-as` 验）
- [ ] 正在进行中的会话（未停服）长按菜单不出现

## FIFO

- [ ] 连续启停 22 次（每次发 1 条消息），主页会话数稳定在 20
- [ ] 其中随意置顶 2 条；再启停 3 次，主页数量为 20 + 2 = 22（置顶不占配额）

## 空会话

- [ ] 启动服务、不发任何消息和文件 → 点停止 → 主页不新增条目

## 崩溃恢复

- [ ] 启动服务发 1 条消息 → 强杀 APP（`adb shell am force-stop com.example.flikky`）→ 重启 → 主页能看到那条会话（endedAt 已补；不是 "进行中"）
- [ ] 同样场景但零消息 → 重启后会话消失

## 文件持久化

- [ ] 浏览器上传文件后停服 → 进入历史 → 再次打开该文件能正常打开
- [ ] 手机推送文件后停服 → 浏览器再次以历史会话恢复访问？（v1.2 功能，v1.1 不验）

## 安全（回归 v1.0）

- [ ] PIN 单次使用、3 次锁 30 秒、5 次终止 —— 与 v1.0 一致

## 稳定性

- [ ] APP 切后台 10 分钟：前台服务仍在
- [ ] 100 MB 文件上传：完成；会话结束后 totalBytes 约等于 100 MB
